# 大本营战报文风

## 口吻

昭和官腔 / 战报体。常用套语：

- 「大本营发表」
- 「据确报」「确报称」
- 「敌军」「残敌」「溃灭」「土崩瓦解」
- 「战果赫赫」「损失轻微」「士气空前」
- 「战线已全面推进」「万全之局业已确立」

避免：现代互联网梗、表情符号、轻松口语、真实战争暴行细节、人身攻击。

## 夸张手法

| 原文倾向 | 战报夸张 |
|----------|----------|
| 大概能行 | 业已奏功 |
| 两小时 / 很快 | 瞬息之间 |
| 有点风险 / 风险不大 | 敌军已无抵抗 |
| 应该没问题 | 万全之局业已确立 |
| 先不管边界 | 残敌不足为患 |
| 差不多写完了 | 圆满克复全境 |

偏悲观或失败时：仍可写成「战略转进圆满」「损失轻微之辉煌撤退」，保持讽刺而非幸灾乐祸。

## 标题

- 长度约 **16–28** 字（中文）
- 一条 dispatch **一个**主标题
- 硬、短、像公报，不要问句

**好例**

- 我军圆满完成重构，敌军复杂度业已溃灭
- 测试防线全线奏功，缺陷军团已无抵抗
- 部署瞬息告捷，线上敌军望风披靡

**坏例**

- 重构成功啦！！！（口语、标点滥用）
- 关于本次代码重构工作的阶段性胜利说明（太长、太公文现代）

## 导语（lede）

1–2 句。建议以「大本营发表：」起笔，复述战果并点明「损失轻微」或「确报」。

## 对照（必须）

`source_claim` 尽量保留原话。喜剧感来自 **官宣 headline/lede vs 前线实报 source_claim** 的落差，不要抹掉原文。

## irony_tag 选用

`速胜` | `无损` | `必然` | `万全` | `转进` | `克复`

择一最贴切者，勿自造长句标签。
